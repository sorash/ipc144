 IPC144

Part C - Modules

Library Functions

Workshop 6


In this workshop, you are to write a simple game that accepts user input and uses random values. 


Learning Outcome

Upon successful completion of this workshop, you will have demonstrated the abilities to

  • validate user input
  • use standard library functions


Dice Throws

Write a program that 'throws' a pair of dice until the sum of the values on their top faces equals a specified number.  The values on the six faces of each die range from one to six inclusive.  Your program prompts for and accepts the number sought, validates the input, requests new new input if current input is invalid and uses the rand() and srand() library functions. 

Code your validation function so that you can use it without modification in another application. 

The output from your program looks like:  

 Game of Dice
 ============
 Enter total sought : 13
 ** Invalid Input! Try Again! **
 Enter total sought : 11
 Result of throw  1 : 1 + 3
 Result of throw  2 : 4 + 4
 Result of throw  3 : 6 + 2
 Result of throw  4 : 5 + 6
 You got your total in 4 throws!

Card Draw (Difficult)

Write a program that 'draws' cards from a deck of 52 cards until it has drawn the card the user is seeking.  Your program prompts for and accepts the suit and the rank of the card sought, request new input if the current input is invalid and uses the rand() and srand() library functions.  You may assume that the user returns the drawn card to the deck.  (If you wish to make your coding slightly more realistic and hence more complicated, you may assume that the user does not return the drawn card to the deck.)

Design your validation functions so that you can use them without modification in another application. 

The output from your program looks like:  

 Card Draw
 =========
 Enter Suit : Q
 ** Invalid Input! Try Again. **
 Enter Suit : H
 Enter Rank : 4
 Result of draw  1 : 2C
 Result of draw  2 : 3S
 Result of draw  3 : KD
 Result of draw  4 : 4H
 You drew your card in 4 draws!
